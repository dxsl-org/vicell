## Overview

Create a minimal ViCell Cell that uses `viui-build` to auto-generate `Counter` from `counter.vi` at build time, then constructs the widget tree at runtime.

Goal: prove the full pipeline works — `.vi` → build-time Rust generation → no_std Cell compile.

## `cells/apps/viui-demo/build.rs`

```rust
fn main() {
    viui_build::compile("*.vi");
}
```

The build script runs on the host. `viui_build::compile` will:
1. Find `counter.vi` in the crate root
2. Emit `cargo:rerun-if-changed=counter.vi`
3. Write generated Rust to `$OUT_DIR/counter.rs`

## `cells/apps/viui-demo/src/main.rs`

```rust
#![no_std]
#![no_main]

extern crate alloc;
extern crate ostd;

// Generated by viui-build from counter.vi
include!(concat!(env!("OUT_DIR"), "/counter.rs"));

api::declare_syscalls![Log];

#[no_mangle]
pub fn main() {
    let (state, _root) = Counter::build();

    // Verify reactivity works — increment count via signal
    state.count.update(|n| *n += 1);

    ostd::syscall::sys_exit(0);
}
```

Notes:
- `extern crate alloc;` required because generated code uses `alloc::format!` etc.
- `#![no_main]` + `#[no_mangle] pub fn main()` is the standard ViCell Cell entry point
- `_root` discards the widget tree (no renderer in P05 — just prove compilation)
- `api::declare_syscalls![Log]` matches other simple demo cells

## Verification Approach

P05 verifies at the **type-check / compile level**, not at runtime (Cell binaries run on RISC-V hardware/emulator, not on the dev host).

Run order:
```powershell
# 1. Verify viui-build compiles on host
cargo check --manifest-path tools/viui-build/Cargo.toml

# 2. Type-check the demo cell (riscv64 target, no_std)
cargo check -p viui-demo

# 3. Full build — produces the Cell ELF
cargo build -p viui-demo
```

**Expected**: all three commands succeed.

If `cargo build` fails due to linker/target issues unrelated to viui-demo code, `cargo check` success is sufficient to declare P05 done — the goal is to prove the generation pipeline works, not to boot the Cell.

## Risk: Generated code imports `viui::node_widgets::row::Row`

The current codegen emits `use viui::node_widgets::row::Row;` unconditionally.  
If `libs/viui/src/node_widgets/row.rs` doesn't exist yet, this would be a compile error.

**Mitigation**: Check if `row.rs` exists before implementing. If missing, emit it (trivial mirror of `column.rs`) or scope the import to only used widget types. The codegen can be updated to only emit imports for types actually used.
